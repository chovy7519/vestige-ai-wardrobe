# Vestige AI Wardrobe

Vestige AI Wardrobe 是一个面向个人穿搭管理的移动端产品原型。它的核心不是简单整理衣服，而是帮助用户沉淀“曾经穿得好看的自己”，并在每天早晨快速复用、调整和生成穿搭方案。

当前版本是一个高保真静态前端原型，重点验证产品结构、交互路径和界面质感；AI 识别与生图能力以产品流程和前端状态进行模拟，尚未接入真实模型服务。

## 产品定位

用户痛点：

- 每天早上花时间想穿什么，决策疲劳明显。
- 相册里的穿搭记录不完整，好的搭配很难被系统复用。
- 衣橱中的单品缺少利用率、搭配关系和穿着成本管理。
- 想尝试替换某件衣服时，不希望反复真实试穿、拍照、上传。

产品目标：

- 把“穿搭照片”转化为结构化衣橱资产。
- 记录高分穿搭记忆，并在合适场景下推荐复穿。
- 支持 AI 识别单品、确认入库、生成替换预览。
- 用清晰、克制、成熟的移动端界面降低早晨决策成本。

## 核心功能

### 今日推荐

- 根据天气、日程、形象强度给出今日首选穿搭。
- 提供“早晨三选”，方便在稳妥、正式、轻松之间快速切换。
- 支持调整今日偏好，并刷新推荐结果。

### 衣橱资产

- 以单品资产为核心管理衣橱。
- 支持按外套、内搭、下装、鞋包筛选。
- 支持搜索颜色、场景、品类等关键词。
- 展示成套组合、核心单品、日均成本等资产指标。

### 穿搭记忆

- 记录历史高分穿搭。
- 以日历和时间线方式查看复穿档案。
- 支持查看穿搭详情、复穿分和组成单品。

### AI 工作台

- 模拟从穿搭照片中识别单品。
- 支持对识别结果逐项确认并入库。
- 支持选择衣橱中的替换单品，生成替换后的穿搭预览。
- 明确区分“风格参考预览”和真实试穿效果，避免过度承诺。

### AI 录入流程

- 支持三类录入入口：穿搭照、单品、购物截图。
- 根据入口切换不同的识别内容、保存策略和确认结果。
- 提供上传、确认、入库三步流程。
- 支持保存穿搭记忆、补充衣橱单品、保存购物截图候选资产。

## 当前技术实现

项目目前使用原生前端实现：

- `HTML`：移动端产品结构与语义化界面。
- `CSS`：高保真视觉样式、响应式手机壳、bottom sheet、卡片和控件系统。
- `JavaScript`：页面导航、筛选搜索、AI 模式切换、录入流程、弹层状态和 toast 反馈。
- `assets/`：穿搭与衣橱场景图片资源。

无需构建工具，直接打开 `index.html` 即可预览。

```bash
open index.html
```

如果希望通过本地服务访问：

```bash
python3 -m http.server 8080
```

然后访问：

```text
http://127.0.0.1:8080
```

## 产品边界

当前版本重点验证“产品是否成立”和“界面是否足够成熟”，不包含以下能力：

- 真实用户账户系统。
- 真实照片上传与云端存储。
- 真实 AI 视觉识别模型。
- 真实虚拟试衣或图像生成服务。
- 后端数据库与衣橱资产持久化。

后续如果进入工程化 MVP，需要补充：

- 前端框架：React / Next.js 或 Expo React Native。
- 后端服务：Node.js / Python API。
- 数据库：PostgreSQL + 对象存储。
- AI 能力：图像识别、服饰属性抽取、穿搭推荐、局部替换生图。
- 账号与隐私：用户照片、衣橱数据和生成结果的权限管理。

## 后续路线

1. 将静态原型迁移为组件化前端。
2. 接入真实图片上传和本地衣橱数据存储。
3. 用视觉模型识别单品类别、颜色、材质、季节和风格标签。
4. 建立穿搭记忆评分和复穿推荐规则。
5. 接入生成模型，验证“替换单品预览”的真实可用性。
6. 增加用户隐私、数据导出、失败纠错和模型置信度提示。

## 设计原则

- 首页优先给出可穿方案，而不是让用户继续整理数据。
- 衣橱不是列表，而是可复用的个人资产层。
- AI 只在识别、确认、替换和推荐中提供辅助，不替用户做不可解释的决定。
- 所有生成预览都应被视为风格参考，不能替代真实试穿判断。
